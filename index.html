ðŸ“– Usage Guide

            
The data-on-keys attribute allows you to bind keyboard events to Datastar actions with powerful key combinations and modifiers.

            
            
Basic Syntax

            data-on-keys[:key-spec][__modifier1][__modifier2]="action"
            
            
Key Specifications

            

                
The key-spec defines which keys trigger the action. It can be:

                
    
                    
  • Single keys: space, enter, escape, tab, f1, a
    • 
                    
  • Key combinations: ctrl-k, alt-enter, shift-space, meta-s, ctrl-shift-z
    • 
                    
  • Multiple options: enter.tab, escape.f1.ctrl-q (any of these will trigger)
    • 
                    
  • Any key: omit the key-spec entirely to capture all key presses
    • 
                

            

            
            
Key Name Mapping

            

                
Common key aliases are automatically mapped to standard JavaScript key names:

                

                    

                        
Special Keys

                        
    
                            
  • space â†’ Space character
    • 
                            
  • enter / return â†’ Enter
    • 
                            
  • escape / esc â†’ Escape
    • 
                            
  • tab â†’ Tab
    • 
                            
  • backspace â†’ Backspace
    • 
                            
  • delete / del â†’ Delete
    • 
                        

                    

                    

                        
Navigation Keys

                        
    
                            
  • up â†’ ArrowUp
    • 
                            
  • down â†’ ArrowDown
    • 
                            
  • left â†’ ArrowLeft
    • 
                            
  • right â†’ ArrowRight
    • 
                            
  • pageup â†’ PageUp
    • 
                            
  • pagedown â†’ PageDown
    • 
                        

                    

                

                

                    Note: Function keys (f1-f12), letter keys (a-z), and other standard keys work as expected.
                

            

            
            
Modifiers

            
            
Target Modifiers

            

                
    
                    
  • Default (no modifier) - Binds to window (global)
    • 
                    
  • __el - Binds to the element itself (requires focus)
    • 
                

            

            
            
Behavior Modifiers

            

                
    
                    
  • Default - Prevents default browser behavior (preventDefault())
    • 
                    
  • __noprevent - Allows default browser behavior
    • 
                    
  • __stop - Stops event propagation (stopPropagation())
    • 
                

            

            
            
Event Timing

            

                
    
                    
  • Default - Triggers on keydown
    • 
                    
  • __up - Triggers on keyup
    • 
                

            

            
            
Event Listener Options

            

                
    
                    
  • __capture - Use capture phase
    • 
                    
  • __passive - Passive event listener
    • 
                    
  • __once - Remove listener after first trigger
    • 
                

            

            
            
Timing Modifiers

            

                
Control when and how often callbacks are executed:

                
    
                    
  • __delay.{time} - Postpone execution by a fixed time
                        
      
                            
    • Example: __delay.500ms or __delay.1s
      • 
                        
    
                    
    • 
                    
  • __debounce.{time} - Wait for user to stop before executing
                        
      
                            
    • Options: __leading, __notrailing
      • 
                            
    • Example: __debounce.300ms__leading
      • 
                        
    
                    
    • 
                    
  • __throttle.{time} - Limit execution rate
                        
      
                            
    • Options: __trailing, __noleading
      • 
                            
    • Example: __throttle.1s__trailing
      • 
                        
    
                    
    • 
                

                

                    Note: Time can be specified in milliseconds (ms) or seconds (s).
                

            

            
            
View Transitions

            

                
    
                    
  • __viewtransition - Wrap state changes in the View Transitions API for smooth animations
                        
      
                            
    • Example: data-on-keys:space__viewtransition="$counter++"
      • 
                        
    
                    
    • 
                

                

                    Note: View Transitions API support varies by browser. The modifier is safely ignored in unsupported browsers.
                

            

            
            
Common Examples

            

                
<!-- Global shortcuts -->
<div data-on-keys:ctrl-s="saveDocument()"></div>
<div data-on-keys:escape="closeModal()"></div>

<!-- Element-specific -->
<input data-on-keys:enter__el="submitForm()">
<textarea data-on-keys:ctrl-enter__el="submit()"></textarea>

<!-- Allow default behavior -->
<div data-on-keys:tab__noprevent="logTab()"></div>

<!-- Multiple triggers -->
<div data-on-keys:escape.f1.ctrl-h="showHelp()"></div>

<!-- Timing modifiers -->
<div data-on-keys:enter__delay.500ms="$message = 'Delayed!'"></div>
<input data-on-keys__debounce.300ms__el="$search($event.target.value)">
<div data-on-keys:space__throttle.1s="$counter++"></div>

<!-- View transitions -->
<div data-on-keys:space__viewtransition="$counter++"></div>

ðŸ’¡ Timing Modifier Tips

                    
    
                        
  • Delay: Postpones execution by a fixed time. Use __delay.500ms or __delay.1s
    • 
                        
  • Debounce: Waits for user to stop before executing. Great for search inputs. Supports __debounce.500ms__leading and __debounce.500ms__notrailing
    • 
                        
  • Throttle: Limits execution rate. Perfect for scroll or resize handlers. Supports __throttle.500ms__trailing and __throttle.500ms__noleading
    • 
                        
  • View Transition: Wraps state changes in the View Transitions API for smooth animations
